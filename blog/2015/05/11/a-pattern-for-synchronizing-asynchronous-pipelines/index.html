A pattern for synchronizing asynchronous pipelines

Working with AVFoundation, I’ve devised a functional pipeline that takes CMSampleBuffers and does things with them. This pipeline was asynchronous, ensuring we’re getting maximum throughput out of it with little regard for memory usage. The problem however, was that the concurrency led to resulting samples coming out of the pipeline in a different order. There is a need to have some sort of a FIFO sync in order to ensure I wasn’t sacrificing accuracy for speed.
